## Why

The extension already checks GitHub Releases for newer versions, but it announces updates through browser notifications that take users away from the product and do not provide a clear download path. Moving update awareness into the popup makes the update available at the moment users are interacting with ChamaLead, while release changelogs make each version understandable and safer to install.

## What Changes

- Replace the update notification flow with an in-popup update notice when a newer GitHub Release is available.
- Surface the latest release changelog in the popup so users can understand what changed before downloading.
- Add a direct download action for the release ZIP asset generated by GitHub Actions.
- Store update metadata from GitHub Releases so the popup can render update state without duplicating release-fetching logic.
- Improve the GitHub Actions release flow and project guidance so future releases include useful changelogs for popup display.
- Remove the need for browser notification permission if no other feature uses it.

## Capabilities

### New Capabilities
- `release-update-flow`: Detect GitHub Release updates, expose update metadata to the popup, let users download the release ZIP, and display release changelogs.

### Modified Capabilities
- None.

## Impact

- `src/extension/background.ts`: update check scheduling, GitHub Release parsing, storage of update metadata, and popup-facing runtime messages.
- `src/pages/popup/PopupPage.tsx`: update notice UI, changelog preview/display, and download/check actions.
- `src/styles/global.css`: styles for the update notice and changelog presentation.
- `vite.config.ts`: extension permissions may change from `notifications` to `downloads` and/or `alarms` depending on implementation choices.
- `.github/workflows/build.yml`: release generation should remain compatible with the popup and produce useful assets/changelogs.
- `AGENTS.md`: development guidance should require changelog-quality release notes for user-visible updates.
